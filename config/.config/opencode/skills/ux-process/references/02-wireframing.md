# Wireframing

**Source**: `ux-designer` skill

## Purpose

Create low-fidelity layouts to validate information architecture and flow before committing to visual design.

## Key Principles

### 1. Low Fidelity First

- Start with simple boxes, lines, and placeholders
- No colors, no fonts, no images
- Focus on layout, spacing, and hierarchy

### 2. Information Architecture

- Group related content together
- Establish clear visual hierarchy
- Place important elements in prominent positions

### 3. Mobile-First

- Design for the smallest screen first
- Identify what content is essential vs. nice-to-have
- Progressive enhancement for larger screens

### 4. Iterate Fast

- Create multiple variations of the same screen
- Test different layouts and information groupings
- Get feedback early before investing in visual design

## Wireframe Types

| Type | Fidelity | When to Use |
|------|----------|-------------|
| **Sketch** | Very low | Initial ideation, quick exploration |
| **Low-fi** | Low | Validating layout with stakeholders |
| **Mid-fi** | Medium | Detailing interactions and states |

## Output

- Screen layouts (boxes + lines)
- Component placement
- Content hierarchy
- Responsive breakpoints
