# User Research

**Source**: `ux-designer` skill

## Purpose

Understand who the users are, what they need, and how they behave before designing solutions.

## Key Activities

### 1. User Interviews

- Talk to real users about their problems and workflows
- Ask open-ended questions: "Tell me about the last time you..."
- Listen for pain points, workarounds, and unmet needs

### 2. Persona Development

- Create user personas based on research data
- Include: goals, frustrations, technical skill level, context of use
- Use personas to validate design decisions

### 3. Competitive Analysis

- Study similar products and their solutions
- Identify patterns users already expect
- Note gaps and opportunities for differentiation

### 4. User Journey Mapping

- Map the complete flow from trigger to goal completion
- Identify friction points and drop-off risks
- Validate assumptions with real user testing

## Output

- User personas
- Journey maps
- Problem statements
- Requirements list
