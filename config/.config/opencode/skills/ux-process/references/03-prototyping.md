# Prototyping

**Source**: `ux-designer` skill

## Purpose

Create interactive models to validate flows, interactions, and user experience before building.

## Key Activities

### 1. Flow Validation

- Test if users can complete key tasks
- Identify confusing navigation patterns
- Validate information flow between screens

### 2. Interaction Testing

- Test hover states, transitions, and animations
- Validate micro-interactions (loading, success, error)
- Ensure feedback is immediate and clear

### 3. User Testing

- Watch real users interact with the prototype
- Don't explain — observe where they struggle
- Record where users hesitate or make mistakes

### 4. Iteration

- Fix identified issues immediately
- Test the fix with the same or different user
- Repeat until the flow is smooth

## Prototype Fidelity

| Fidelity | Tool | When to Use |
|----------|------|-------------|
| **Paper** | Pen + paper | Initial concept validation |
| **Low-fi** | Figma/wireframe tools | Layout and flow validation |
| **Mid-fi** | Interactive wireframes | Interaction and timing validation |
| **Hi-fi** | Code or advanced tools | Final UX validation before build |

## Output

- Interactive prototype
- User testing notes
- Identified friction points
- Validated user flows
