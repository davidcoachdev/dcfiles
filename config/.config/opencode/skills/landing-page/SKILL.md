## When to Use

- Building a landing page for a SaaS, app, or service
- Rewriting an existing page that doesn't convert
- User says "make a landing page" without specifying details
- Need a single-offer page (one intent, one audience, one action)

## Core Concept

A landing page is NOT a homepage.
A homepage serves multiple intents.
A landing page wins ONE intent: **one offer → one audience → one primary action**.

## Before You Start — Gather These

If the user gives vague instructions, ask:

### Page Purpose
- What's the ONE primary action? (trial, demo, buy, waitlist, download)
- What's the offer exactly?
- What counts as conversion? (click, signup, purchase)

### Audience + Context
- Who's the ICP?
- What problem are they solving?
- Top 3 objections (why they don't convert today)
- Traffic source: ads / search / social / email

### Proof + Assets
- Logos, testimonials, numbers, case studies
- Screenshots, demo video, product GIFs
- Guarantees / refund / cancellation terms

### Constraints
- Brand voice: casual vs professional
- Design direction: minimal editorial vs glass UI vs playful 3D
- Mobile priority?

## Core Structure

### Above the Fold (mandatory)
1. **Headline** — outcome + audience
2. **Subheadline** — clarifies "how" with specificity
3. **Primary CTA** — clear verb + what they get
4. **One proof signal** — logo strip / stat / short testimonial
5. **Hero visual** — product screenshot/video or strong illustration

### Mid Page (the argument)
6. **Problem → solution** — one section
7. **Benefits** — 3–5 outcome-driven
8. **How it works** — 3 steps
9. **Social proof** — testimonials / case study

### Bottom (objection handling)
10. **FAQ** — 6–12 Q&A
11. **Risk reversal** — trial, cancel anytime, guarantee
12. **Final CTA** — same as top

## Layout Types — Pick One

| Type | Best When | Example |
|------|-----------|---------|
| **A) Classic hero + sections** | Product understandable with a screenshot | Most SaaS landing pages |
| **B) Long-form story (sales page)** | Need to educate + overcome skepticism | High-ticket courses, consulting |
| **C) Minimal conversion page** | High-intent traffic (email → known users) | Download, waitlist |
| **D) Comparison page** | Search intent includes alternatives ("X vs Y") | SEO-driven landing pages |

## Conversion Strategies (practical)

### 1. Match Message to Source
If traffic is from ads: mirror the ad headline in the hero. Same promise, same visual tone.

### 2. Make the Next Step Obvious
- One primary CTA above the fold
- No competing CTAs (no "Learn more" next to "Start free trial")

### 3. Write Benefit-First
- **Features**: what it does
- **Benefits**: what that means for them
- ❌ "Save time and streamline"
- ✅ "Cut your weekly reporting from 4 hours to 15 minutes"

### 4. Use Specificity
Numbers, timeframes, concrete outcomes. Vagueness kills conversion.

### 5. Reduce Risk
Pick at least one: free trial, free plan, no credit card, cancel anytime, money-back guarantee.

### 6. Handle Objections Early
FAQ is NOT a footer afterthought. Put it where friction is highest. Put proof next to the claim it supports.

## Copywriting Templates

### Headline Formulas
- `"{Outcome} without {pain}"`
- `"The {category} for {audience}"`
- `"Ship {result} in {time}"`

### Subheadline Rules
- 1–2 sentences
- Clarify what it is + who it's for

### CTA Rules
- Verb + what they get
- Avoid weak CTAs: "Learn more", "Submit"
- Examples: "Start free trial", "Book a demo", "Get the checklist"

### Benefit Bullets
Format: **Benefit** — proof/detail
Example: **Faster iteration** — generate 3 layout variants in one click.

## Build Order (section by section)
1. Hero
2. Benefits
3. How it works
4. Proof
5. FAQ
6. Final CTA

**Rule**: Don't rebuild the whole page each time. Iterate section-by-section to keep control.

## SEO + AEO Checklist

### When NOT to Index
- Ad-only campaigns, highly time-bound offers
- Use `noindex`

### When TO Index
- Evergreen offers, search intent matches promise
- Add: clear title + meta, internal links from homepage, FAQ in plain Q&A for AEO

## Common Pitfalls
- Too many CTAs above the fold
- Vague value prop ("streamline", "optimize")
- Big feature list with no outcomes
- Proof hidden at the bottom
- Mobile layout breaks readability
- No clear next step
- Designing for what looks good instead of what converts

## Output Format

When generating a landing page, return:
1. **Page outline** — sections + order
2. **Hero copy** — headline, subheadline, CTA, proof line
3. **Benefits section** — 3–5 bullets
4. **How it works** — 3 steps
5. **FAQ** — 6–12 Q&A
6. **SEO/AEO** — indexing recommendation + title/meta if indexed
7. **Layout recommendation** — A/B/C/D + why

## Quick Questions

If user says "make me a landing page":
- What's the ONE primary CTA?
- Who's the ICP and what's the main pain?
- Any proof (numbers/testimonials/logos)?
- What's the offer and risk reversal?
- Where is traffic coming from?
