## Los 23 Patrones

### Creacionales (5)

| Pattern | Descripción |
|---------|-------------|
| Singleton | Una sola instancia con acceso global |
| Factory Method | Interfaz para crear objetos, subclases决定 tipo |
| Abstract Factory | Familia de objetos relacionados sin especificar clases concretas |
| Builder | Construir objetos complejos paso a paso |
| Prototype | Clonar objetos existentes |

### Estructurales (7)

| Pattern | Descripción |
|---------|-------------|
| Adapter | Interfaz incompatible → compatible |
| Bridge | Separar abstracción de implementación |
| Composite | Árbol de objetos con interfaz uniforme |
| Decorator | Agregar features dinámicamente |
| Facade | Simplificar sistema complejo |
| Flyweight | Muchos objetos similares资源共享 |
| Proxy | Controlar acceso a objeto |

### Comportamiento (11)

| Pattern | Descripción |
|---------|-------------|
| Observer | Suscripción y notificación de cambios |
| Strategy | Intercambiar algoritmos dinámicamente |
| State | Comportamiento según estado interno |
| Command | Request como objeto (cola, undo) |
| Chain of Responsibility | Cadena de handlers |
| Iterator | Recorrer colección sin exponer estructura |
| Mediator | Reducir acoplamiento entre objetos |
| Memento | Guardar/restaurar estado interno |
| Template Method | Algoritmo con pasos definidos |
| Visitor | Agregar operaciones a jerarquía |
