# ER / Data Model

**Best for:** entities, fields, relationships, database schema.

## Layout
- Entities: rectangles with header + fields
- Fields: name + type
- PK: key icon
- FK: italic or dashed
- Relationships: crow's foot notation
- Cardinality: 1:1, 1:N, N:M

## Anti-patterns
- Too many entities (max 8)
- All fields shown → show only key fields
- Relationships without cardinality
