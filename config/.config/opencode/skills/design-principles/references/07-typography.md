# Typography

**Tips 10-16**: Typography rules for UI.

## Type Selection

- Use a **single sans serif** typeface — most legible, neutral, simple for UI
- Avoid mixing typefaces — it adds visual noise
- Choose typefaces with **taller lowercase letters** (larger x-height = better readability at small sizes)
- Example: Inter has better x-height than League Spartan

## Font Weights

- Use **regular and bold only**
- Avoid thin, light, semibold weights in UI
- Reserve extreme weights for large headings only
- Bold for headings, regular for body text

## Text Formatting

- **Limit uppercase** — forces reading letter-by-letter (all rectangles)
- Use **sentence case** (only first word + proper nouns capitalized)
- Exception: very short labels where shape recognition matters less

## Text Color

- **Avoid pure black** (#000000) — causes eye strain (100% brightness difference)
- Use dark grey instead (e.g., `#333333` or `#1a1a1a`)
- Adjust grey lightness based on importance (lighter = less prominent)

## Text Alignment

- **Left align** — English reads left-to-right, F-shaped pattern
- Left alignment gives consistent starting point per line
- Avoid center-aligned body text — makes reading harder
- Center alignment OK for short headings only

## Line Height

- Use at least **1.5 line height** for body text
- Range 1.5-2.0 is optimal for readability
- Prevents rereading same line
- Makes text feel comfortable and professional

## Quick Reference

| Rule | Value |
|------|-------|
| Typeface | Single sans serif |
| Weights | Regular (400) + Bold (700) only |
| Case | Sentence case |
| Text color | Dark grey, never pure black |
| Alignment | Left (headings can be center) |
| Line height | ≥1.5 for body text |
