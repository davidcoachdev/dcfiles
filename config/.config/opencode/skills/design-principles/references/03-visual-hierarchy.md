# Visual Hierarchy

**Tip 4**: Create a clear visual hierarchy.

## Rules

- Present information in order of importance
- More important = more prominent (size, color, contrast, spacing, position)
- **Squint Test**: squint/blur your eyes — most important element should stand out
- Primary action should be most prominent element on the interface

## The Squint Test

1. Step back from screen
2. Squint or blur your eyes
3. What stands out first?
4. If it's NOT the primary action → hierarchy is wrong

## Hierarchy Tools (by strength)

1. **Size** — bigger = more important
2. **Color** — saturated = draws attention
3. **Contrast** — high contrast = prominent
4. **Spacing** — isolation = importance
5. **Position** — top/left = noticed first
