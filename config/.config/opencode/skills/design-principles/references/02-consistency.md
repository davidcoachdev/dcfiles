# Consistency

**Tips 2, 3**: Be consistent. Similar-looking elements must function similarly.

## Rules

- Similar elements must look AND work similarly
- Match internal product patterns AND established platform conventions
- Icon styles: use same stroke weight (e.g., 2pt), same corner treatment
- Add text labels to icons for accessibility (screen readers)
- **If elements look similar, people expect them to work similarly**
- Conversely: elements with different functionality should look different
- Remove misleading visual styles (e.g., buttons that aren't clickable)

## Anti-Patterns

- ❌ Button that looks like a link
- ❌ Link that looks like a button
- ❌ Non-interactive element styled as interactive
- ❌ Mixed icon stroke weights in same UI
- ❌ Different border-radius for same component type

## Consistency Checklist

| Check | Question |
|-------|----------|
| Visual | Do similar elements look the same? |
| Behavioral | Do similar elements work the same way? |
| External | Do we match platform conventions? |
| Icons | Same stroke weight? Same corner treatment? |
| Labels | Do icons have text labels for accessibility? |
